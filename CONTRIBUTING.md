# Contributing to QuickPalette

Thank you for your interest in contributing to QuickPalette! This document provides guidelines and information about contributing to this project.

### Reporting Bugs

If you find a bug, please report it by opening an issue on GitHub. Please include:

- A clear description of the problem
- Steps to reproduce the issue
- Expected behavior vs. actual behavior
- Screenshots if applicable
- Information about your browser and operating system

### Suggesting Enhancements

We welcome suggestions for new features and improvements! Please:

- Check if the enhancement has already been suggested
- Open an issue with a clear description of the proposed enhancement
- Explain why this enhancement would be valuable
- Provide any implementation ideas if you have them

### Pull Requests

We welcome pull requests! Here's how to contribute:

1. Fork the repository
2. Create a new branch for your feature or bug fix: `git checkout -b feature-name`
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes with a clear commit message
6. Push to your fork: `git push origin feature-name`
7. Open a pull request

When you submit a pull request:

1. Provide a clear description of your changes
2. Reference any related issues
3. Include screenshots if your changes affect the UI
4. Ensure your PR passes any automated checks
5. Be responsive to feedback and make requested changes

## ğŸ·ï¸ Commit Message Guidelines

Use clear and descriptive commit messages:

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally after the first line

Examples:
```
Add color palette export functionality
Fix drag and drop on mobile devices
Update dependencies to latest versions
```
